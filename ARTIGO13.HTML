Como Criar um Site Grátis com GitHub Pages

📅 08 Mar 2026 ⏱ 8 min de leitura
EL
Eng. Lamego Desenvolvedor Web & Consultor Digital — Tete, Moçambique

Ter um site profissional já não é exclusivo de quem tem dinheiro para pagar hosting e domínios caros. O GitHub Pages é um serviço completamente gratuito que permite publicar um site profissional na internet em menos de 30 minutos — sem pagar um único centavo. É exactamente o serviço que uso para hospedar este site.

Neste guia vou mostrar-te passo a passo como criar e publicar o teu site usando o GitHub Pages, desde a criação da conta até ao site estar online e acessível para qualquer pessoa no mundo.

O GitHub Pages é usado por milhões de desenvolvedores e empresas em todo o mundo para hospedar sites, portfólios e documentação — completamente de graça.— GitHub Documentation

O que é o GitHub Pages

O GitHub é uma plataforma onde programadores guardam e partilham código. O GitHub Pages é uma funcionalidade que transforma os ficheiros HTML, CSS e JavaScript do teu repositório num site público acessível através de um URL do tipo username.github.io. É gratuito, fiável e sem limites de tráfego para sites estáticos.

O que precisas antes de começar

📋 Requisitos

  • Conta de email (Gmail recomendado)
  • Ligação à internet
  • Os ficheiros do teu site (HTML, CSS, imagens)
  • 15 a 30 minutos de tempo

Passo 1 — Criar conta no GitHub

Acessa github.com e clica em "Sign up". Preenche o email, cria uma password segura e escolhe um nome de utilizador — este nome vai fazer parte do teu URL, por isso escolhe algo profissional. Por exemplo, se o teu nome de utilizador for joaosilva, o teu site ficará em joaosilva.github.io.

Confirma o email que recebes na caixa de entrada e o teu registo fica completo.

Passo 2 — Criar o repositório

Após entrar na tua conta, clica no botão verde "New" ou vai a github.com/new. No campo "Repository name" escreve exactamente: username.github.io — substituindo "username" pelo teu nome de utilizador real. Este nome exacto é obrigatório para o GitHub Pages funcionar correctamente.

Marca o repositório como Public e clica em "Create repository".

Passo 3 — Fazer upload dos ficheiros

Dentro do repositório criado, clica em "uploading an existing file". Arrasta todos os ficheiros do teu site (HTML, CSS, imagens) para a área de upload. Certifica-te que o ficheiro principal se chama index.html — é este que o GitHub vai mostrar como página inicial.

Escreve uma mensagem de commit como "Primeiro upload do site" e clica em "Commit changes".

Passo 4 — Activar o GitHub Pages

Vai às Settings do repositório (engrenagem no topo). No menu lateral clica em "Pages". Em "Source" selecciona "Deploy from a branch", escolhe o branch main e pasta / (root). Clica em "Save".

Passo 5 — O site está online!

Aguarda 2 a 5 minutos e acessa username.github.io no browser. O teu site está publicado e acessível para qualquer pessoa no mundo — de graça, para sempre.

✅ Vantagens do GitHub Pages

  • Completamente gratuito — sem custos de hosting
  • HTTPS incluído — site seguro automaticamente
  • Sem limites de tráfego para sites estáticos
  • Fácil de actualizar — basta fazer upload dos ficheiros novos
  • URL profissional: username.github.io

Dica extra — Domínio personalizado

Se quiseres um domínio próprio como www.meusite.co.mz em vez de username.github.io, podes configurá-lo gratuitamente nas Settings do GitHub Pages. Só precisas de comprar o domínio (cerca de 500-1000 MZN por ano) e apontar os DNS para o GitHub. O hosting continua gratuito.
